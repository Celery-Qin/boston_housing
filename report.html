编程练习3：定义衡量标准¶

如果不能对模型的训练和测试的表现进行量化地评估，我们就很难衡量模型的好坏。通常我们会定义一些衡量标准，这些标准可以通过对某些误差或者拟合程度的计算来得到。在这个项目中，你将通过运算决定系数 R2 来量化模型的表现。模型的决定系数是回归分析中十分常用的统计信息，经常被当作衡量模型预测能力好坏的标准。

R2的数值范围从0至1，表示目标变量的预测值和实际值之间的相关程度平方的百分比。一个模型的R2 值为0还不如直接用平均值来预测效果好；而一个R2 值为1的模型则可以对目标变量进行完美的预测。从0至1之间的数值，则表示该模型中目标变量中有百分之多少能够用特征来解释。模型也可能出现负值的R2，这种情况下模型所做预测有时会比直接计算目标变量的平均值差很多。

在下方代码的 performance_metric 函数中，你要实现：

  • 使用 sklearn.metrics 中的 r2_score 来计算 y_true 和 y_predict的R2值，作为对其表现的评判。
  • 将他们的表现评分储存到score变量中。

或

  • (可选) 不使用任何外部库，参考决定系数的定义进行计算，这也可以帮助你更好的理解决定系数在什么情况下等于0或等于1。

from sklearn.metrics import r2_score

def performance_metric(y_true, y_predict):
    """计算并返回预测值相比于预测值的分数"""

    score = r2_score(y_true, y_predict)

    return score

# 不允许导入任何计算决定系数的库

def performance_metric2(y_true, y_predict):
    """计算并返回预测值相比于预测值的分数"""
    y_true = pd.Series(y_true)
    y_predict = pd.Series(y_predict)
    total_ss = ((y_true - y_true.mean()) ** 2).sum()
    res_ss = ((y_true - y_predict) ** 2).sum()
    score = 1 - (res_ss / total_ss)

    return score

问题 8 - 回答：¶

  • 什么是K折交叉验证法（k-fold cross-validation）？

就是将训练集平均分配到k个容器中，取其中一个作为验证容器，剩下（k-1）个作为训练容器。这样依次重复k次，最后将这k次的测试结果取平均值。

  • GridSearchCV是如何结合交叉验证来完成对最佳参数组合的选择的？

我的理解是，先进行网格搜索，在参数的各种排列组合中，计算每一组的得分。

在计算每一组得分的时候，结合交叉验证，最大化利用训练集数据。

最后得出得分最高的那一组参数组合。

  • GridSearchCV中的'cv_results_'属性能告诉我们什么？

'cv_results_'能告诉我们不同参数组合交叉验证的结果，具体指标有：

def cross_val_score(estimator, X, y, scoring = performance_metric, cv=3):
    """ 返回每组交叉验证的模型分数的数组 """
    scores = [0,0,0]
    test_size = int(len(X) / cv)
    for i in range(cv):
        X_test = X.iloc[(i*test_size):((i+1)*test_size-1),:]
        X_train = X.drop(X_test.index.tolist(), axis=0)
        y_test = y[(i*test_size):((i+1)*test_size-1)]
        y_train = y.drop(y_test.index.tolist(), axis=0)
        estimator = estimator.fit(X_train, y_train)
        y_pred = estimator.predict(X_test)
        scores[i] = scoring(y_test, y_pred)        
    return scores

def fit_model2(X, y):
    """ 基于输入数据 [X,y]，利于网格搜索找到最优的决策树模型"""
    max_score = 0
    best_max_depth = 0
    for i in range(1, 11):
        regressor = DecisionTreeRegressor(max_depth=i)
        scores = cross_val_score(regressor, X, y)
        score = sum(scores) / len(scores)
        if score > max_score:
            max_score = score
            best_max_depth = i
    
    #最优交叉验证分数对应的最优模型
    best_estimator = DecisionTreeRegressor(max_depth=best_max_depth)
    
    return best_estimator

问题 13 - 实用性探讨¶

简单地讨论一下你建构的模型能否在现实世界中使用？

提示：回答以下几个问题，并给出相应结论的理由：

问题 13 - 回答：¶

  • 1978年所采集的数据，在已考虑通货膨胀的前提下，在今天可能不太能适用。因为时代不同，民众对于房屋房间数量、社区贫困指数、附近的教育资源的看法可能有所改变。比如当今交通更发达，可能附近的教育资源有限，但人们可以通过先进交通工具来获取远距离的优质教育资源等。
  • 数据中呈现但特征也远远不足以描述一个房屋，其他重要特征还有诸如：交通、商业、医疗、自然环境、房屋新旧程度等。
  • 在波士顿这样的大都市采集的数据，不能应用在其它乡镇地区。因为不同地区的人，对房屋特征的看重程度不同，比如乡镇地区人口密度可能比较低，贫富差距小，从而人们对社区贫困指数不是很看重。
  • 仅仅凭房屋所在社区的环境来判断房屋价值是不合理的，还应该考虑如房屋新旧程度、房屋面积、层高、户型等诸多因素。
